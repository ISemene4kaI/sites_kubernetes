# Sites kubernetes

## Краткое описание

Репозиторий нужен для удобной компановки, двух сайтов и более в один kubernetes кластер, чтобы не делать большое кол-во серверов.
Суть в том, чтобы поставить это всё на один сервер, поставить балансер нагрузок и использовать все сайты.
Сайты мелкие одностраничники, не создают сильной нагрузки.

## Что используется в проекте

Kubernetes - как основной инструмент для связи Deployment Pod и Ingress Controller
Docker - контейнеризация приложений/сайтов.
Ingress Controller - Traefik, не используем Nginx, т.к. он считается устаревшим на момент 2026 года. В новых проектах советуется не брать его в пример.
Argo CD - инструмент для автоматической доставки обновлений в kubernetes кластер из репозиториев.

Репозиториии приложений: [репо1](https://github.com/ISemene4kaI/about_me) [репо2](https://github.com/ISemene4kaI/semka_informatics)

## Принцип работы проекта

Есть 3 репозитория, пример структуры
```
Github
- sites_kubernetes
- site1
- site2
...
```

Мы используем первый репозиторий `sites_kubernetes` в качестве связующего звена между репозиториями сайтов.
В нём мы пишем всю логику деплоя, сборки и развёртывания kubernetes кластера. В эту логику включаем Argo CD для сбора обновлений git репозиториев, как достоверный источник данных.
После сбора постепенно внедряем обновления без падения сайта в ошибки 5xx.

## Принцип работы Ingress Controller

Ingress Controller должен будет работать под одному принципу, если на сервер приходит http/https запрос он смотрет на домен и перенаправляет в соответствующий Pod с нужным нам сайтом
